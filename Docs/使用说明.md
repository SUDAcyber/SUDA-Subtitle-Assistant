# SUDA字幕翻译助手使用说明

## 首次打开

本软件目前没有 Apple Developer ID 签名和公证。将 App 拖入“应用程序”后，在 Finder 中按住 Control 点击 App，选择“打开”；若仍被阻止，请前往“系统设置 → 隐私与安全性”点击“仍要打开”。不需要关闭 Gatekeeper。

## AI 翻译接口配置

SUDA 不附带任何 API Key。API Key 是模型服务商用于鉴权和计费的密钥，网页会员订阅通常不等于 API 额度。密钥只保存在当前 Mac 的 Keychain，不会写入项目、DMG 或字幕文件。切换服务商后，请同时换成该服务商对应的 Key。

| API 来源 | 选择 | Base URL | 配置方法 |
| --- | --- | --- | --- |
| OpenAI 官方 | OpenAI 官方 | 自动填写 `https://api.openai.com/v1` | 在 `https://platform.openai.com/api-keys` 创建 Key，复制可用模型 ID |
| Anthropic 官方 | Claude 官方 | 自动填写 `https://api.anthropic.com` | 在 `https://console.anthropic.com/settings/keys` 创建 Key，复制可用 Claude 模型 ID |
| OpenRouter | OpenRouter | 自动填写 `https://openrouter.ai/api/v1` | 在 `https://openrouter.ai/settings/keys` 创建 Key，从模型页复制带厂商前缀的模型 ID |
| OpenAI 兼容中转 | 中转服务 | 按平台 API 文档填写 | 使用中转平台自己的 Key 和模型 ID |
| 火山方舟（豆包） | 中转服务 | `https://ark.cn-beijing.volces.com/api/v3` | 使用火山方舟 API Key；模型通常填写推理接入点 ID `ep-...` |

### 中转地址怎么判断和填写

1. 文档必须明确写有“OpenAI 兼容”“OpenAI-compatible”或兼容 Chat Completions。
2. 填 API 根地址，不要填平台官网、控制台或登录地址。
3. 如果文档的完整请求地址是 `https://example.com/v1/chat/completions`，SUDA 中通常只填 `https://example.com/v1`，软件会自动追加 `chat/completions`。
4. API Key、Base URL 和模型 ID 必须来自同一个服务商。模型 ID 要逐字复制，不能只填“GPT”“Claude”或“豆包”等展示名称。
5. 普通中转、OpenRouter 和豆包优先使用“聊天补全”。OpenAI 官方只有在模型明确支持时才使用 Responses。Claude 官方自动走 Anthropic Messages。
6. 仅提供 Anthropic/Gemini 原生协议、签名鉴权、专用 SDK 或额外必填请求头的平台，不能只靠地址和 Key 接入“中转服务”。

### 常见接口错误

- `401/403`：Key 错误、Key 与地址不属于同一家服务、余额不足或账户无模型权限。
- `404`：Base URL 缺少或重复 `/v1`、误填了完整的 `chat/completions` 路径，或模型 ID 不存在。
- `429`：余额不足、请求过快或平台限流；降低并发请求后重试。
- `400`：模型不支持当前接口模式、推理参数或 JSON 输出；先改用聊天补全和平台明确支持的模型。

字幕原文、剧情分析样本和本地添加的翻译记忆会发送给所选模型服务商。处理敏感内容前，请阅读该服务商的隐私政策。使用云端 API 会产生费用，长字幕、上下文分析、重试和较高并发都会增加消耗。

## 语音识别

- Typhoon ASR：泰语本地识别，无需 `.env`、API Key 或 Hugging Face Token。在“设置 → 语音识别”点击一键安装，文件位于 `~/Library/Application Support/SUDA字幕翻译助手/typhoon/`。
- WhisperKit：公开模型无需 Key，首次使用会下载模型；`large-v3` 约 3.1GB，识别时约占 2–3GB 内存。
- ElevenLabs Scribe：需要 ElevenLabs API Key，音频会上传并按账户规则计费；不希望上传音频时请选择本地识别。

模型识别结果仅供参考。泰语背景音乐、多人抢话、口音和收音质量都可能造成错字或时间轴误差，导出前需要人工校对。

## 导入、导出和 MKV

支持 SRT 及常见 MP4、MOV、M4V、MP3、WAV、M4A、AAC、FLAC、AIFF、CAF 等文件。MKV 不受 macOS AVFoundation 直接支持，请先用 ffmpeg 转为 MP4，或提取为 M4A/WAV。

“导出原字幕”保留识别原文和时间轴；“生成字幕/另存为”优先导出译文。查找替换的“定位下一处”可循环跳转并高亮匹配字幕。
